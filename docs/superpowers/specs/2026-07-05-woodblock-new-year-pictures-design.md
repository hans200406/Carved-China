# 《版上山河》课程设计规格

## 1. 项目定位

- 课程主题：数字遗珍——非遗文化多维信息可视化设计。
- 作品题目：《版上山河——中国传统木版年画多维信息可视化》。
- 研究范围：全国传统木版年画代表性流派对比，以杨柳青、桃花坞、杨家埠、朱仙镇、绵竹为核心，按数据可得性补充其他国家级项目。
- 技术路线：HTML、CSS、原生 JavaScript、ECharts、p5.js；无需构建工具即可本地运行，并支持 GitHub Pages 静态部署。
- 核心叙事：从年画的视觉吸引进入全国分布和历史数据，再由宏观分析过渡到题材、色彩与纹样的微观生成艺术，最终落到保护与传承。

## 2. 验收目标

作品必须同时满足四个模块：

1. 模块 A（40%）：基于真实数据完成准确的数据图表，并明确写出至少三条有效洞察。
2. 模块 B（40%）：不直接堆砌照片，将年画题材、色彩和纹样矢量化、几何化并通过 p5.js 生成艺术重构。
3. 模块 C（10%）：将数据与艺术整合为一个滚动叙事网页，包含悬停、点击、筛选和滚动入场等交互。
4. 模块 D（10%）：提交不少于 2000 字的设计说明文档，覆盖背景、数据来源与清洗、图表依据、视觉生成逻辑和交互说明，并附过程草图。

## 3. 数据设计

### 3.1 数据范围与来源

优先使用中国非物质文化遗产网、中国政府网、地方文旅厅、地方非遗保护中心及博物馆公开资料。百科类资料只作为线索或补充，不作为关键统计结论的唯一依据。每条数据保留来源名称、URL、访问日期和备注。

### 3.2 数据字段

- 项目名称、标准流派名称、所在省市、经纬度
- 起源年代、历史阶段、国家级名录批次
- 代表题材、文化寓意、主要纹样
- 主要色彩名称、HEX 色值、色彩角色
- 主要制作工序、代表性特征
- 数据来源、访问日期、可信度备注

### 3.3 清洗规则

- 统一行政区和流派名称，原始值另列保留。
- 模糊年代转换为可排序的历史阶段，同时保留原始文字。
- 题材统一归入门神、吉祥祈福、娃娃仕女、戏曲故事、花鸟瑞兽、民俗生活等类别。
- 颜色统一为中文色名和 HEX；推断色值必须标注方法，不能伪装为官方统计。
- 去重、缺失值、异常值和人工补充步骤记录在清洗说明中。

## 4. 模块 A：数据可视化

### 4.1 图表组合

1. 全国分布地图：用地理点位和流派筛选展示年画项目的空间格局。
2. 历史时间轴：比较主要流派形成、发展和保护节点。
3. 题材矩形树图：比较各流派代表题材的数量或整理频次。
4. 色彩关系图：以分组色带、堆叠条形或关系图比较流派主色组合。

图表必须具有标题、单位、图例、悬停提示、数据来源和必要注释，不使用 ECharts 默认配色。

### 4.2 洞察方向

- 年画流派的形成与区域民俗、商业城镇和交通传播存在关联。
- 各流派在题材功能和叙事偏好上存在地域差异。
- 红、黄、绿等高纯度传统色具有共性，但配色比例和组合结构形成流派辨识度。

以上仅为待验证假设；最终洞察必须由采集后的数据计算支持，不预设结论。

## 5. 模块 B：生成艺术

### 5.1 视觉元素

将门神、莲花、鲤鱼、娃娃、花鸟、云纹和边框纹样拆分为原创 SVG 或 Canvas 几何图元。不得把网络图片直接作为生成作品主体。

### 5.2 生成规则

- 对称结构模拟门画和窗花式构图。
- 图元密度、旋转、缩放和重复由确定性随机种子控制。
- 流派切换改变图元组合、边框节奏和色板。
- 题材切换改变中心符号和辅助纹样。
- 用户可调整密度并点击“重新套印”生成新构图。
- 支持导出当前生成画面为 PNG，便于过程记录和海报制作。

## 6. 模块 C：页面与交互

### 6.1 页面结构

1. 开场：题目、导语和套色揭纸动效。
2. 年画中国：全国地图与项目概览。
3. 百年流变：历史时间轴。
4. 图必有意：题材结构和文化寓意。
5. 五色成章：流派色彩比较。
6. 版上生花：p5.js 生成艺术工作台。
7. 守艺新生：三条洞察、数据局限与保护结语。

### 6.2 交互与状态

- 导航支持锚点跳转和当前章节提示。
- 地图点击流派后，题材与色彩图表同步筛选。
- 所有图表提供悬停说明；筛选器支持重置。
- 滚动动画只触发视觉入场，不隐藏关键内容。
- 数据或依赖加载失败时显示可读的错误提示与静态摘要。
- 支持 `prefers-reduced-motion`，减少动画模式下保持完整信息。
- 手机端不依赖悬停，点击可获得同等信息。

## 7. 视觉规范

视觉体系参考 `E:/awesome-design-md-main/awesome-design-md-main/design-md/claude/DESIGN.md` 的暖色编辑式布局、设计令牌和响应式原则，但不复制其品牌资产。

- 画布：宣纸暖白；正文为墨黑；主强调色为朱砂红；辅助色为藤黄和石绿。
- 标题：思源宋体或系统宋体回退；正文和控件：思源黑体、微软雅黑或系统无衬线回退。
- 最大内容宽度约 1200px，使用 4px 基础间距系统，主要章节留白 80–96px。
- 页面节奏交替使用宣纸浅色、深墨色和朱砂强调章节。
- 卡片使用细边框、轻微纸张色差、0–12px 圆角和极弱阴影。
- 动效以 300–600ms 缓动模拟套色、揭纸与叠印；禁止霓虹、玻璃拟态和无意义大面积渐变。
- 桌面、平板、手机三档布局；触控目标不小于 44px。

项目根目录需建立独立 `DESIGN.md`，把上述原则落实为颜色、字体、间距、组件、图表和响应式令牌。

## 8. 技术结构

- `index.html`：语义化页面结构。
- `styles/`：设计令牌、基础样式、组件、响应式规则。
- `scripts/`：入口、图表、筛选联动、滚动交互和 p5.js 生成艺术。
- `data/raw/`：原始数据和来源记录，只增不改。
- `data/processed/`：清洗后的 CSV/JSON。
- `tools/`：可重复执行的数据清洗与报告构建脚本。
- `docs/`：清洗说明、过程草图、设计报告源文件及规格。
- `assets/`：原创 SVG、字体说明和必要的本地静态资源。
- `poster/`：海报源文件和最终 PNG。
- `dist/` 或根目录静态文件：可直接运行/部署版本。

模块之间通过统一的标准化 JSON 数据和共享筛选状态通信。图表、生成艺术和页面交互彼此独立，任何单模块失败不应阻断其他内容阅读。

## 9. 验证方案

- 数据：核对记录数、去重、必填字段、分类枚举、HEX 格式及来源完整性。
- 功能：验证全部图表、筛选联动、导航、生成艺术参数、PNG 导出和错误状态。
- 浏览器：验证桌面与手机尺寸，检查控制台错误、资源加载和触控可用性。
- 可访问性：检查标题层级、键盘焦点、颜色对比、替代文本和减少动画设置。
- 文档：设计报告不少于 2000 字；DOCX/PDF逐页渲染检查，无截断、重叠或乱码。
- 海报：精确为 1080×1920，文字清晰且与网页视觉一致。
- 提交：在干净目录解压最终 ZIP，按 README 指引验证可运行性和文件齐全性。

## 11. 范围边界

- 不使用 Vue/React、后端数据库或需要复杂安装的构建链。
- 不承诺获取不存在的官方产值或传承人年度统计；数据不足时调整图表，不编造数据。
- 不以照片墙代替视觉重构；外部图片若用于背景介绍，必须注明来源和版权状态。
- 在线部署需要可用的 GitHub/Vercel 账号或由用户自行完成最终授权步骤；本地可运行版本不受此限制。
