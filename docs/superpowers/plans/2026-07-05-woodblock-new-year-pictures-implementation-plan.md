# 《版上山河》实施计划

## 总体原则

- 工作目录固定为 `E:/作业/信息可视化设计/23120032038`。
- 每个阶段先定义验收条件，再制作，再验证并提交 Git 版本。
- 关键统计结论必须能追溯到原始数据和来源，不能用推测替代数据。
- 网页、文档、海报共享同一套设计令牌和内容口径。
- 先完成最小可用闭环，再增加动效和视觉润色，避免最后只剩“漂亮但无法验收”的页面。

## 阶段 1：项目骨架与视觉系统

### 工作内容

1. 建立 `index.html`、`styles/`、`scripts/`、`data/raw/`、`data/processed/`、`assets/`、`tools/`、`docs/`、`poster/` 和 `vendor/`。
2. 从参考库提炼专用 `DESIGN.md`，明确颜色、字体、间距、组件、图表及响应式令牌。
3. 创建页面七个叙事章节和基础导航。
4. 配置本地静态服务器说明，并确保不依赖联网 CDN 也能运行核心内容。

### 验收

- 本地可打开页面，无控制台错误。
- 桌面和手机均能浏览完整章节。
- 所有新增文件均位于指定目录。

## 阶段 2：真实数据采集与清洗

### 工作内容

1. 从中国非物质文化遗产网获取国家级木版年画项目、类别、地区和批次。
2. 从政府、文旅厅、保护中心和博物馆资料补充主要流派的起源年代、题材、工序和色彩。
3. 保存原始抓取或人工整理表，建立 `sources.csv` 来源台账。
4. 编写可重复执行的清洗脚本，输出标准 CSV 与网页 JSON。
5. 记录去重、行政区标准化、年代归一、题材分类和色值整理过程。

### 验收

- 每条关键记录有来源 URL 和访问日期。
- 原始数据只读保留，清洗结果可由脚本重建。
- 自动检查重复记录、空值、分类和 HEX 色值。
- 不存在无法解释的统计数字。

## 阶段 3：模块 A 数据分析可视化

### 工作内容

1. 实现全国分布地图和流派点击筛选。
2. 实现历史时间轴。
3. 实现题材矩形树图。
4. 实现流派传统色彩比较图。
5. 建立共享筛选状态，使地图选择同步影响题材和色彩图表。
6. 根据清洗数据计算并撰写至少三条洞察，标注样本范围和局限。

### 验收

- 图表类型与数据变量匹配。
- 标题、图例、单位、悬停说明和来源齐全。
- 筛选、重置和窗口缩放正常。
- 三条洞察可由代码或清洗数据复算。

## 阶段 4：模块 B 生成艺术

### 工作内容

1. 设计门神、莲花、鲤鱼、娃娃、花鸟、云纹和边框的原创矢量图元。
2. 通过 p5.js 实现对称、重复、套色偏移和确定性随机布局。
3. 提供流派、题材、色板、密度和随机种子控制。
4. 实现“重新套印”和 PNG 导出。
5. 整理纹样拆解、色彩映射和算法逻辑，供报告使用。

### 验收

- 不依赖直接堆砌照片。
- 相同种子生成相同结果。
- 所有控件键盘、鼠标和触控可用。
- 导出图片尺寸和内容正确。

## 阶段 5：模块 C 叙事与交互整合

### 工作内容

1. 串联开场、地图、时间轴、题材、色彩、生成艺术和保护结语。
2. 添加滚动入场、套色显现和章节导航反馈。
3. 实现加载失败、数据为空和依赖失败状态。
4. 完成桌面、平板和手机响应式适配。
5. 支持减少动画设置，并为非悬停设备提供等价点击交互。

### 验收

- 滚动叙事连续，无生硬页面跳转。
- 基础交互至少覆盖悬停、点击和滚动。
- 关闭动画后信息不缺失。
- 页面无横向溢出、遮挡和不可操作控件。

## 阶段 6：模块 D 文档与过程材料

### 工作内容

1. 制作数据清洗过程说明 DOCX/PDF。
2. 制作不少于 2000 字的设计说明 DOCX/PDF，包含：
   - 项目背景与选题意义
   - 数据来源、字段与清洗方法
   - 四类图表的选择依据和数据洞察
   - 纹样提取、色彩体系和生成算法
   - 整体叙事、交互和技术实现
   - 局限性、版权与数据声明
3. 制作页面线框、图表草图、纹样拆解和配色推导过程图并插入报告。
4. 生成目录、图注和参考资料列表。

### 验收

- 正文不少于 2000 字。
- 数据口径与网页一致。
- DOCX 与 PDF 逐页渲染检查，无乱码、截断或重叠。
- 过程草图真实对应实现过程。

## 阶段 7：海报与在线版本

### 工作内容

1. 制作 1080×1920 竖版课程展览海报。
2. 海报包含作品名称、核心视觉、主要图表、三条洞察、生成纹样和作者信息占位。
3. 生成网页部署版本及部署说明。
4. 若具备账号授权，部署至 GitHub Pages 或 Vercel；否则保留可直接部署包和清晰步骤。
5. 制作在线访问 URL 说明文档。

### 验收

- 海报像素精确为 1080×1920。
- 海报在100%比例下文字清晰、层级明确。
- 在线或本地版本从干净环境可运行。
- URL、二维码或本地说明不指向失效地址。

## 阶段 8：最终审计与提交包

### 工作内容

1. 按任务书和评分表建立逐项验收矩阵。
2. 检查数据真实性、三条洞察、生成艺术、交互、报告字数和海报尺寸。
3. 检查 HTML 语义、键盘操作、颜色对比、响应式、控制台和资源路径。
4. 清理临时文件；保留源文件、依赖、数据、文档、报告、海报和说明。
5. 在新目录外的临时位置解压并验证最终 ZIP，确认相对路径可用。
6. 根据用户提供的学号和姓名生成规范文件名：`学号_姓名_非遗课设.zip`。

### 最终文件清单

- `README.md`
- `DESIGN.md`
- 完整网页源码及本地依赖
- 生成艺术源文件
- 原始数据、来源台账、清洗数据和清洗脚本
- 清洗过程说明 DOCX/PDF
- 设计说明文档 DOCX/PDF
- 过程草图与图表导出
- 1080×1920 海报 PNG 及源文件
- 在线访问 URL 说明文档
- 最终 ZIP

## 执行顺序与依赖

数据是模块 A、报告和海报的共同上游，因此必须先于正式图表定稿。生成艺术可以在数据清洗后半段并行设计，但最终色板和流派选项必须以标准数据为准。报告从数据阶段开始同步记录，避免项目结束后反向编造过程。海报最后制作，以使用通过验收的图表和生成纹样。

## 需要用户补充的信息

最终打包前需要确认学号、姓名、报告是否有学校固定模板，以及在线部署账号授权。缺少这些信息不影响前六个阶段实施。
