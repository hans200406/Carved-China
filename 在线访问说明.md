# 在线访问说明

项目已经包含 GitHub Pages 自动部署配置，但当前本地仓库尚未绑定远程 GitHub 仓库，因此暂时没有真实在线地址。

## 本地访问

双击 `启动项目.cmd`，程序会启动本地服务器并尝试在 Microsoft Edge 中打开：

`http://127.0.0.1:8038`

## GitHub Pages 发布

1. 在 GitHub 创建一个空仓库。
2. 将本项目推送到该仓库的 `master` 或 `main` 分支。
3. 在仓库 Settings → Pages 中将 Source 设为 GitHub Actions。
4. 等待 `Deploy static course project to GitHub Pages` 工作流完成。
5. 将工作流输出的页面地址填写到最终提交材料中。

根据用户要求，海报不放置二维码。
